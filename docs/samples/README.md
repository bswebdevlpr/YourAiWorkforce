# Golden Samples

Real, end-to-end artifacts produced by the phase-0 agents — committed as evidence that the
pipeline actually runs and produces coherent output on a local 8B model.

## What these are (and aren't)

- **`*/prd.md` is raw, unedited output.** The document text was generated by the
  `product_discovery` agent (`qwen3:8b`, local Ollama). Not a single line of the body was
  hand-edited — including model artifacts (see below). That's the point: it's evidence, not a
  brochure.
- **My only input was the conversation itself.** Driving phase-0 is a human-in-the-loop dialog.
  I played the "founder" role — supplying the idea and answering the agent's clarifying questions
  (Action, personas, keywords, approvals). That steering is normal HITL input; I did not write or
  edit the resulting artifact.
- **`*/prd.en.md` is a human translation** of the Korean original, for readability only. It is
  *not* system output. The authoritative artifact is always the Korean `prd.md`.

## Known artifacts in this run (left in on purpose)

Honesty about the local-8B ceiling — these are real and visible in the trace, not hidden:

- **Hallucinated date** — the `## 작성일` (date) field reads `2023-10-15`, invented by the model.
- **Mid-conversation marker leaks** — in a couple of turns the agent echoed its own persona
  instruction lines (`🛑 ** ... get user approval`) into the reply. Note: these appear in the
  *conversation*, **not** in the final saved `prd.md` — the save-validation gate keeps the artifact clean.
- **One self-corrected hallucination** — the agent briefly pulled "character battle simulation"
  from a persona example, then corrected itself the next turn after I pushed back.

The **saved artifact itself is clean**: all six sections filled, personas consistently mapped to
features and metrics, zero leftover template placeholders. The rough edges live in the conversation
UX, which is exactly where the post-processing work continues.

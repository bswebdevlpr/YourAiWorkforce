## 프로젝트명
한자를 게임처럼 학습하는 모바일 앱

## 작성일
2023-10-15

## 작성자
기획자

## 1. 문제 정의
- **Action**: 학습 (한자를 외우고 읽는 능력 익히기)
- **Domain Object**: 한자
- **Platform**: 모바일 앱
- **핵심 문제**: 반복 학습 지루함, 복습 시기 간과, 부수/획순 규칙 복잡성

## 2. 타겟 사용자
### 페르소나 1: 김민수 (17세, 고3)
- 배경: 고등학교 3학년, 한자능력검정 8급 준비 중
- 기술 수준: 디지털 리터러시 중간
- 목표: 한자능력검정 1급 합격 및 대학 입시 활용
- 페인 포인트: 반복 학습 지루함, 복습 시기 간과
- 사용 시나리오: 수업 후 15분간 10개 카드 복습, 데일리 스트릭 동기부여

### 페르소나 2: 박지현 (28세, 회계사)
- 배경: 중국어 문서 처리 필요, 한자 의사소통 장벽
- 기술 수준: 모바일 앱 활용 능력 높음
- 목표: 한자 읽기 능력 향상, 문화적 이해 증대
- 페인 포인트: 교재 단조로운 형식, 시간 부족
- 사용 시나리오: 출퇴근 시 10분 간 간격반복복습, 레벨업 경쟁

### 페르소나 3: 이준호 (35세, 일본어 학습자)
- 배경: 일본어 JLPT N4 준비 중, 한자 기초 학습 필요
- 기술 수준: 전문가 수준
- 목표: 한자 필기 인식 및 뜻·음 매칭 능력 강화
- 페인 포인트: 부수/획순 규칙로 인한 학습 지연
- 사용 시나리오: 아침 커피 시간 15개 카드 학습, 게이미피케이션 동기부여

## 3. 기능 우선순위
### P0 (MVP 필수)
1. SRS 간격반복복습 시스템: 한자능력검정 8급~1급 기준 자동 조정 (대상: 김민수, 박지현, 이준호)
2. 필기 인식 퀴즈: 한자 쓰기 실력 평가 및 피드백 (대상: 이준호)
3. 데일리 스트릭 시스템: 7일 연속 학습 시 보상 제공 (대상: 김민수, 박지현)
4. 게이미피케이션 카드 수집: 레벨업 시 스타일링 요소 추가 (대상: 박지현, 이준호)
5. 부수/획순 규칙 학습: 한자 구성 요소별 분류 및 연습 (대상: 이준호)

### P1 (Critical)
6. 한자능력검정 목표 설정: 8급~1급 단계별 학습 플랜 생성 (대상: 김민수)
7. 뜻·음 매칭 퀴즈: 음독 훈련 및 실시간 피드백 (대상: 박지현, 이준호)
8. 학습 성과 시각화: 학습 시간/정확도 통계 제공 (대상: 김민수, 이준호)

### P2 (Nice-to-have)
9. 공부 친구 초대 기능: 경쟁/협업 모드 추가 (대상: 박지현)
10. 문화적 맥락 학습: 한자 유래 이야기/문화 배경 제공 (대상: 박지현)

### P3 (Future)
11. AI 개인화 학습 추천: 사용자 패턴 기반 맞춤형 콘텐츠 (대상: 이준호)
12. 멀티 언어 지원: 일본어/중국어 자막/퀴즈 추가 (대상: 이준호)

## 4. 성공 지표
1. 일일 학습 시간: 20분/일 (대상: 김민수, 박지현, 이준호) - 측정: 데일리 스트릭 기록, 임계값: 14일 이상 지속
2. SRS 복습 정확도: 85% 이상 (대상: 김민수, 이준호) - 측정: 필기 인식 퀴즈 결과, 임계값: 75% 미만 시 복습 간격 조정
3. 레벨업 달성률: 30일 내 5개 레벨업 (대상: 박지현, 이준호) - 측정: 게이미피케이션 이력, 임계값: 20일 이내 달성 시 추가 보상
4. 한자 읽기 정확도: 90% 이상 (대상: 박지현, 이준호) - 측정: 뜻·음 매칭 퀴즈, 임계값: 80% 미만 시 부수/획순 추가 학습
5. 학습 성과 공유: 50% 이상 사용자 참여 (대상: 모든 페르소나) - 측정: 공부 친구 초대 기능, 임계값: 30% 미만 시 인센티브 재설계

## 5. 범위 경계
### In Scope
- 한자능력검정 급수 기반 학습
- SRS 간격반복복습
- 필기 인식 및 뜻·음 매칭 퀴즈
- 데일리 스트릭 시스템
- 게이미피케이션 카드 수집/레벨업
- 부수/획순 규칙 학습

### Out of Scope
- 소셜 미디어 통합
- 외부 API 연동 (예: 교육 플랫폼)
- 고급 AI 맞춤형 추천

## 6. Phase 마일스톤 (예상)
- **Phase 0 완료**: PRD + 아키텍처 + 도메인 지식 문서
- **Phase 1 완료**: 데이터베이스 + 기본 API + 빌드 설정
- **Phase 2 완료**: 모든 P0 기능 작동
- **Phase 3 완료**: P1 기능 + 알고리즘 구현
- **Phase 4 완료**: SEO + 성능 최적화 + i18n
- **Phase 5 완료**: 도메인 전문가 검증 통과
- **Phase 6 완료**: 프로덕션 배포
